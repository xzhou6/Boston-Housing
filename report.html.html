Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Answer: Grid search is an exhaustive searching through a manually specified subset of the hyperparameter space of a learning algorithm. With this method, we try every combination of the provided hyper-parameter values to identify the best model. Then we evaluate each combinations performance, typically using cross-validation. We choose the best cross-validation accuracy that matches with the corresponding parameters to optimize the learning algorithm. Scikit-learn provides grid search method for algorithm parameter tuning. This approach has its drawbacks. Higher-dimensional hyperparametric spaces are far more time consuming to test than the simple two-dimensional problem. Also, with a fixed number of training samples for any given model, the models predictive power decreases as the number of dimensions increases (Hughes phenomenon).

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer: K-fold cross validation training technique is a method to estimate a tuning parameter in the model. The dataset is randomly divided into k equal sized subsets. Each time, one of the k subsets is used as the testing set and the other k-1 subsets are put together to form a training set. Then the average error across all k trials is computed. Benefit of this technique provide for grid search when optimizing a model: While doing grid search with K-fold cross validation, we are able to evaluate the model performance more reliably. K-fold cross validation allows for multiple testing subsets and hence the evaluation does not rely on any particular data subset, which enables us to see whether the test performance varies based on which data subset you use to train/test. The variance of the resulting estimate is reduced as the number of K increases. It gives us the average error rate which represents generalization error better. K-Fold cross validation process iterates k times until every subset have been used as the testing set. It matters less how the data gets divided (think about a unbalanced dataset, a simple split may keep all similar data points together and make training set and testing set completely different). Also, it mitgates the overfitting problem thanks to a larger validation set size. When data sample tends to be small and is not able to be split into training and testing sets without losing significant modelling/testing capability, K-fold cross validation becomes a better alternative.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # ShuffleSplit randomly sample your entire dataset during each iteration to generate a training set and a testing set. 
    # Its test_size and train_size parameters cocntrol how large the test and train set should be for eachc iteration.
    # Since you are sampling from the entire dataset during each iteration, values selected during one iteration could be
    # selected again during another iternation.
    # ShuffleSplit saves computation time when your dataset grows
    # KFOLD divides dataset into pre-specified number of folds, and every data point must be in one and only one fold.
    
    # X.shape[0] is the total number of elements
    # n_iter is the number of re-shuffling & splitting iterations.
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':list(range(1,11))} # last value in range is exclusive
    #dt_range = range(1, 11)
    #params = dict(max_depth=dt_range)
    
    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)
    
    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
